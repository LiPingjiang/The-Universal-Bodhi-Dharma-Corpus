## 每部独立文献的文件体例

每一部独立文献对应 **Excel 一行 + 三份 Markdown**，构成"四件套"：

| 文件 | 命名规则 | 内容定位 |
|---|---|---|
| Excel 行 | 见多级索引 | 元信息：名称、原名、年代、语言、编号、md 路径等 |
| `XX_原文.md` | `经名_原文.md` | 原始语言落盘 + **双轨汉译对照**，存真第一 |
| `XX_解读_学术.md` | `经名_解读_学术.md` | 学术解读：文献考证、术语精确、思想史定位 |
| `XX_解读_白话.md` | `经名_解读_白话.md` | 白话解读：全程现代中文熟词，术语沉入脚注 |

### 原文.md 规范

#### 双轨汉译对照

`XX_原文.md` 的汉译对照部分采用**甲乙双轨**结构，每偈/每段并排呈现：

- **甲译（白话意译）**：通顺自然，不拘泥于原文句式，适合直接阅读。用现代口语，地名、人名等专有名词须在首次出现时加括号说明（如"祇树给孤独园（给孤独长者捐建的精舍，位于舍卫城郊外）"）。
- **乙译（严谨译）**：力求严谨，可用文言、文白夹杂或现代学术汉语，逐句对应原文，术语精确，供对照研究。不要求白话，以忠实为第一原则。

两译共用同一套脚注，不重复定义。

#### 脚注规范

- 脚注标签使用**具名标签**，不用数字序号，避免甲乙两译引用同一概念时编号混乱。
- 命名规则：取巴利/梵文关键词的简写，如 `[^nibbāna]`、`[^saccā]`、`[^brahma]`。
- 脚注内容：术语原文 + 字面本义 + 简要说明 + 附录链接（如有）。

#### 原文落盘规范

- **短经**：原文全录 + 汉译全文对照
- **长经**：录开篇总纲、核心名句、结尾，并注明"全文见 SuttaCentral/CBETA"
- 巴利原文使用拉丁转写（含变音符号：ā ī ū ṭ ḍ ṇ ṃ ḷ）
- 梵文同上；汉文直接录原文；藏文录威利转写或藏文原字

### 解读_学术.md 规范

**定位**：学术性解读，面向有一定基础的读者。

**固定章节结构**（可按实际情况增减）：

1. **文献来源与版本** — 收录位置、校订本、汉译版本情况
2. **题名考释** — 拆解原名词根，还原字面本义，考察文化背景
3. **结构分析** — 全经结构、偈颂/散文分布
4. **内容层次分析** — 逐层解析核心义理
5. **思想史定位** — 在佛教知识体系中的位置（核心硬教义 / 大众实践指南 / 戒律制度 / 后期哲学论辩等）；与婆罗门传统、其他宗教的对话关系
6. **传承与注释** — 主要注疏、现代学术参考文献

术语使用巴利/梵文原词，首次出现时附汉译，如"Maṅgala（吉祥）"。

---

### 解读_白话.md 规范

**定位**：面向无佛学背景的普通读者，流畅如读一篇现代文章。

**写作原则**：

- 正文**不出现**令读者陌生的音译术语（涅槃、般若、三昧、阿罗汉等）
- 一切专名以大白话表达，如"把心里那把火彻底熄灭"而非"证悟涅槃"
- 原始术语以 Markdown 脚注形式保留，并附附录链接
- 结构不强制，保持行文流畅
- 地名、人名等专有名词须在首次出现时说明其含义

## 附录体系

附录目录 `附录/` 下含两个核心文件，互相交叉引用：

| 文件 | 作用 |
|---|---|
| `GLOSSARY.md` | 知识网络枢纽，分五个分区（术语/人物/地名/文词/文化背景），逐条详细考据 |
| `古今熟语对照.md` | 项目理念精华，用五列表格呈现每个词"当时是大白话 → 汉译后变难懂 → 今天怎么说"的**转化过程**，破除翻译隔膜 |

### 古今熟语对照.md 体例

- 五列结构：词汇 ｜ 原词（巴利/梵）｜ 当时听到的感觉（古代大白话）｜ 为何汉译后变难懂 ｜ 今天的大白话
- 核心目的：让今天的中国人能像两千五百年前的巴利人一样，自然听懂佛经词汇的本义
- 按概念类型分组（核心概念/修行心性/人物称谓/文体场景/宗教生态）
- 对最易误解的词（如"天人"）设"特别说明"专条，列出常见误解（❌）与正解
- 与 GLOSSARY.md 双向链接：GLOSSARY 条目指向对照表专条，对照表指回 GLOSSARY 详细考据

### GLOSSARY.md 分区结构

`附录/GLOSSARY.md` 是全藏的知识网络枢纽，分五个分区，统一在一个文件中（保持链接路径稳定）：

| 分区 | 收录内容 | 典型条目 |
|---|---|---|
| **术语** | 佛教专业术语，含巴利/梵文原词、字面本义、佛教含义 | 涅槃、四圣谛、无我、缘起 |
| **人物** | 经典中出现的历史人物，含真名、绰号、各语言对应、生平背景 | 给孤独/须达多、祇陀太子、憍陈如 |
| **地名** | 经典中出现的地名，含古代位置、人文地理、现代对应地点 | 舍卫城、鹿野苑、王舍城、鹿野苑 |
| **文词** | 佛典专用文词（非术语），用熟语转换说明 | 偈颂、如是我闻、世尊、比丘 |
| **文化背景** | 理解经典所需的古印度文化、宗教、社会背景 | 天人/天界宇宙观、婆罗门传统、沙门运动 |

**写作原则：**
- 人物条目须包含：真名（原语言）、绰号/称谓、各语言对应（巴利/梵/汉/英）、身份背景、与佛陀的关系、出现文献
- 地名条目须包含：古代名称（原语言）、地理位置（古代描述）、人文背景、现代对应地点（国家/城市/坐标）、出现文献
- 文化背景条目须包含：当时当地人的理解方式、与佛教的关系（对立/融合/借用）、对理解经典的意义
- 文词条目须包含：字面意思、熟语转换（用现代口语说是什么）、使用场景

**熟语速读规范（重要）：**
- 凡是字面与本义有隔膜、容易望文生义的条目（尤其音译词、含古义的称谓、多人名拼合的地名），应在条目开头加一行 **熟语速读**，用**现代人现代行文风格的大白话**一句话点破"到底是什么"，再在后文括号/分段详细考据
- 熟语翻译必须用真正的现代口语，不能用另一堆古词解释古词（反例：用"长者"解释"给孤独"、用"韵文/诗体段落"解释"偈颂"；正例：直接说"爱做慈善的大富商""能唱出来的诗句段落"）
- 多字拼合的名称（如"祇树给孤独园"）应**逐字拆解**，说明每个字到底指人、指物还是指地
- 主动点破常见误解（如"长者"≠老人、"给孤独"≠孤独本人、"天人"≠上帝、"佛陀"≠高僧），可用 ⚠️ 标记提醒
- **称号当专名的处理**：若某称号在当时语境专指一人（如"佛陀"专指悉达多·乔达摩），熟语翻译应同时给出本名+称号含义（"那位叫悉达多·乔达摩的觉悟者"），并说明称号≠某更宽泛的现代词（佛陀≠高僧：佛陀是靠自己彻底觉悟并开宗立教者，独一份；高僧只是修行好的出家人，有很多位）

**源语言夹注体例（重要）：**
- 凡音译或绰号类条目，**标题括号内**应直接夹注源语言与本义，格式：`### 词条（原词，巴利语/梵语，意为"……"）`
- 示例：`给孤独（Anāthapiṇḍika，巴利语，意为"给孤苦无依者施食的人"）`、`祇陀太子（Jeta，巴利语，意为"胜利者"）`、`佛陀（Buddha，巴利语/梵语，意为"觉醒者"）`
- 此体例让读者在标题处即可秒懂词义来源，无需读完正文

## 贡献规范

- 新增文献请先在对应的 `XX_明细.xlsx` 中添加节点行，再创建对应 md 文件
- 文件命名统一使用中文，不用拼音缩写
- 中间生成脚本（Python 等）在产物验证通过后删除，不提交入库
- Commit message 格式：`feat: 新增 XX经 原文+解释` / `fix: 修正 XX经 术语注释`
